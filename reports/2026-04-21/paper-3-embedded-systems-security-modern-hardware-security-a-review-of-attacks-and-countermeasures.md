## 執行摘要

隨著雲端服務、物聯網（IoT）及智慧裝置的普及，硬體安全已成為網路安全防禦體系的基石。本簡報綜述了當代運算系統面臨的微架構與實體層漏洞，重點探討旁路攻擊（Side-Channel Attacks）、推測執行漏洞（如 Spectre 和 Meltdown）、記憶體安全及硬體故障注入技術。

核心發現指出，硬體安全性正處於動態演變中：

**推測執行漏洞：** 現代處理器為提升性能採取的優化手段（如分支預測）反成為資料洩漏的溫床。

**旁路分析：** 攻擊者透過監測功耗、電磁輻射或緩存計時，無需物理破壞即可提取金鑰。

**RISC-V 的崛起與挑戰：** RISC-V 的開放性與模組化為安全創新提供了土壤，但也因其可擴展性引入了獨特的微架構威脅。

**綜合防禦體系：** 未來的韌性硬體需結合記憶體加密、硬體隔離（安全飛地）、加密指令集擴展（ISA）以及實體不可複製功能（PUF）。

## 一、 微架構旁路攻擊（Cache Side-Channels）

緩存（Cache）旨在縮短 CPU 存取資料的延遲，但其時間特性的差異（命中與缺失）被攻擊者利用以推斷敏感資訊。

### 1.1 主要攻擊技術

**Prime+Probe（填充與探測）：** 攻擊者填充緩存空間，等待受害者執行後，再次探測以發現哪些緩存行被替換，從而推斷受害者的存取模式。

**Flush+Reload（刷新與重載）：** 利用共享記憶體，攻擊者刷新特定緩存行，在受害者執行後重載該行並計時。若存取極快，則代表受害者已存取該資料。

**Evict+Reload（驅逐與重載）：** 類似於 Flush+Reload，但透過緩存驅逐而非特定的刷新指令來實施攻擊。

**Flush+Flush：** 測量執行刷新指令兩次的時間差異。此方法較為安靜，產生的噪音較少。

### 1.2 推測執行威脅

現代 CPU 預先執行未來可能需要的指令以提高效率。若預測錯誤，結果會被丟棄，但其對緩存狀態的影響依然存在。

| 攻擊名稱 | 攻擊機制 | 影響範圍 |
| --- | --- | --- |
| Spectre (變體 1-4) | 利用分支預測器、分支目標緩衝區 (BTB) 或存儲到加載轉發 (STL) 洩漏資料。 | 廣泛存在於現代高效能處理器。 |
| Meltdown | 繞過用戶與核心空間之間的記憶體保護，直接讀取內核記憶體。 | 主要影響 Intel 及部分 ARM 處理器。 |
| LVI (負載值注入) | 將攻擊者提供的數值注入推測路徑，操縱程式內部狀態。 | 針對負載操作的微架構干預。 |
| ZombieLoad / RIDL | 利用內部 CPU 緩衝區（如填充緩衝區）洩漏在傳輸中的敏感資料。 | 跨特權邊界的資料採樣攻擊。 |

## 二、 實體旁路與故障注入攻击

此類攻擊利用硬體執行運算時產生的實體特徵（如功耗、電磁）或人為引入錯誤來破解加密系統。

### 2.1 功耗分析（Power Analysis）

**簡單功耗分析 (SPA)：** 直接檢查單一功耗軌跡，以辨識如 RSA 模冪運算等明顯的指令特徵。

**差分功耗分析 (DPA)：** 統計分析多條功耗軌跡，尋找數據與功耗之間的關聯性，對 AES 與 DES 特別有效。

**相關功耗分析 (CPA)：** DPA 的進階形式，將實際功耗與基於假設金鑰的中間值預測模型進行相關性對比。

**模板攻擊 (Template Attacks)：** 威力最強的手段，需先在相同設備上建立詳細的統計模型。

### 2.2 故障注入（Fault Injection）

**電壓閃變 (Voltage Glitching)：** 短暫降低電源電壓，引發計時錯誤或跳過安全檢查。

**電磁故障注入 (EMFI)：** 透過電磁脈衝干擾設備操作。

**Rowhammer：** 透過重複存取 DRAM 的特定行，引發相鄰行的位元翻轉（Bit Flips）。

## 三、 記憶體安全與加密防禦

記憶體加密是防止實體存取攻擊（如冷啟動攻擊）的關鍵。

**機密性與完整性：**

**Intel TME / AMD SME：** 提供透明的全記憶體加密，無需修改軟體即可抵禦物理記憶體提取。

**Merkle 樹：** 用於驗證大型數據集的完整性，任何數據塊的修改都會改變根雜湊值，從而實現篡改檢測。

**防禦策略：**

**遮罩 (Masking)：** 將敏感數據與隨機值結合，使中間計算結果看起來像隨機噪音。

**更換金鑰 (Re-keying)：** 頻繁更新加密金鑰以減少洩漏風險，常結合層次化金鑰管理系統。

## 四、 安全啟動、根信任與安全飛地

建立系統信任鏈是確保硬體安全的基礎。

**安全啟動 (Secure Boot)：** 驗證啟動鏈中的各個組件（從 BIOS/UEFI 到核心）。

**硬體根信任 (RoT)：** 通常由可信平台模組 (TPM) 實現，安全存儲密鑰並執行完整性測量。

**安全飛地 (Secure Enclaves/TEEs)：**

如 **Intel SGX** 和 **ARM TrustZone**，在處理器內創建隔離區域。

提供硬體級隔離、遠端驗證（Remote Attestation）和數據密封（Data Sealing）。

## 五、 RISC-V 架構的安全性研究

RISC-V 的開放性使其成為安全研究的理想平台，但也面臨特定的挑戰。

### 5.1 針對 RISC-V 的特定緩解方案

**SpecTerminator：** 透過硬體染色追蹤（Taint Tracking）和指令遮罩來分類敏感指令，降低推測旁路風險。

**BasicBlocker：** 修改 RISC-V 指令集架構，以犧牲部分性能為代價徹底消除推測執行攻擊。

**隨機動態頻率縮放 (RDFS)：** 使加密運算時的頻率不可預測，有效對抗相關功耗分析 (CPA)。

### 5.2 加密指令集擴展 (Cryptographic ISAs)

RISC-V 通過引入專用的加密擴展（如 Zkne, Zknh），直接在硬體層面加速 AES、SHA-256 等算法。這不僅提升了性能（如 AES 協處理器可提升 37.9% 的效率），也增強了抵禦旁路攻擊的能力。

## 六、 結論與建議

硬體安全是一個持續演進的領域，單一的防禦措施已不足以應對當前的威脅景觀。有效的防禦體系必須採取多層次的方法：

**設計階段強化：** 在設計初期引入 SpectreCheck 等工具檢測潛在旁路。

**硬體級隔離：** 廣泛應用快取分區（Cache Partitioning）與安全飛地。

**隨機化技術：** 採用隨機快取替換政策、隨機延遲插入及時鐘隨機化。

**持續監測：** 利用機器學習與硬體性能計數器進行實時攻擊檢測。

總之，硬體安全的未來在於學術界與產業界的深度協作，共同建立更具韌性的開放標準與防禦機制。
