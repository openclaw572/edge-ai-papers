# RISC-V 硬體與架構安全研究綜述報告

## 1. 研究背景與動機
隨著物聯網（Internet of Things, IoT）革命的興起，嵌入式處理器的市場需求正迎來爆發式增長。長期以來，該領域主要由 Arm 指令集架構（Instruction Set Architecture, ISA）主導。然而，RISC-V 作為一套免費且開放的標準架構，憑藉其開源、簡潔與可擴充套件性，正展現出強大的架構採用動能，並開始挑戰傳統私有架構的地位。

根據當前技術發展趨勢，RISC-V 的應用已擴及以下三個關鍵裝置領域：
*   **小型物聯網裝置 (Small IoT Devices)：** 如低功耗微控制器與邊緣感測器。
*   **個人行動裝置 (Personal Mobile Devices)：** 用於智慧型手機與平板電腦的複雜運算單元。
*   **倉庫級電腦 (Warehouse-scale Computers)：** 應用於雲端資料中心的高效能伺服器處理器。

## 2. 問題定義與挑戰
資料安全與隱私保護是物聯網生態系統面臨的共同挑戰。物聯網裝置頻繁處理大資料，極易成為資料篡改、隱私洩漏及阻斷服務攻擊的目標。此外，硬體層級的微架構洩漏（Microarchitectural leakage）漏洞（如 Meltdown 與 Spectre）的披露，揭示了安全性必須作為硬體與軟體邊界的系統級約束。

RISC-V 的「開放性」是一把雙面刃：一方面，開放原始碼允許公眾審計，有助於建立更透明的「可信計算基」（Trusted Computing Base, TCB）；另一方面，架構細節的公開也讓攻擊者更容易發現微架構缺陷，進而開發更精準的攻擊載荷。目前學術界對於 RISC-V 安全方案的綜述仍存在缺口，特別是對於最新硬體緩解技術與未來趨勢的系統性分析。

## 3. 研究方法與調查範疇
本報告採用分類研究法，參考 RISC-V 硬體與架構安全光譜圖，將安全技術研究範疇劃分為以下六個關鍵領域：
*   **硬體與實體安全 (Hardware and Physical Security)：** 涵蓋 RTL 漏洞偵測、硬體木馬防護及邏輯鎖定。
*   **硬體輔助安全單元 (Hardware-assisted Security Units)：** 包括程式模糊化器、更換單元與專用加密引擎。
*   **記憶體保護 (Memory Protection)：** 側重於標籤記憶體、物理隔離與加密驗證機制。
*   **ISA 安全擴充套件 (ISA Security Extensions)：** 透過指令集強化控制流完整性（Control Flow Integrity, CFI）及側向通道抗性。
*   **密碼學原語 (Cryptographic Primitives)：** 實現輕量化加密、真隨機數生成（TRNG）及後量子密碼學加速。
*   **側向通道攻擊防護 (Side-channel Attack Protection)：** 針對分支預測、計時通道與電源分析的緩解技術。

## 4. 硬體架構與安全基礎

### 4.1 特權模式 (Privilege Modes)
RISC-V 透過多層特權級別實現軟體隔離，其概念與 Armv8-A 的 Exception Levels (EL) 相似：

| 特權級別 | RISC-V 模式 | Armv8-A 等效 | 功能描述 |
| :--- | :--- | :--- | :--- |
| Level 0 | User (U-mode) | EL0 | 執行非特權應用程式，許可權最低。 |
| Level 1 | Supervisor (S-mode) | EL1 | 執行作業系統核心，負責資源管理與隔離。 |
| Level 3 | Machine (M-mode) | EL3 | 最高許可權，具備硬體直接存取權，是必選的特權級別。 |

*註：RISC-V 原有的 H-mode (Hypervisor) 已併入 S-mode 以支援遞迴虛擬化，確保架構簡潔性。*

### 4.2 根信任 (Root of Trust, RoT) 與硬體安全模組 (HSM)
根信任（RoT）是系統安全執行的基石。硬體安全模組（HSM）通常作為獨立或整合的硬體單元，為系統提供不可篡改的安全服務。

**HSM 核心元件與安全功能：**

| 元件名稱 | 安全功能說明 |
| :--- | :--- |
| **OTP 管理模組** | 存取一次性可程式設計記憶體（One-Time Programmable），儲存 UUID、OEM 金鑰與安全啟動策略。 |
| **TRNG 引擎** | 真隨機數生成器（True Random Number Generator），為密碼協議提供高熵值的熵源。 |
| **AES 引擎** | 對稱加密引擎，確保資料機密性並加速安全啟動中的映像檔解密。 |
| **公鑰引擎** | 執行 RSA 或 ECC 非對稱運算，用於數字簽章驗證與身分認證。 |
| **HASH/HMAC 引擎** | 確保資料完整性，支援 SHA-256 等訊息摘要運算。 |
| **安全記憶體存取** | 提供受物理隔離的 RAM 區域儲存敏感資產，與一般處理器域完全隔絕。 |

## 5. 系統架構流程與關鍵技術

### 5.1 實體記憶體保護 (Physical Memory Protection, PMP)
PMP 是 RISC-V 極具競爭力的安全單元，允許 M-mode 透過暫存器控制低許可權模式對實體記憶體的 RWX 許可權。

**RISC-V PMP 與 ARM MPU 技術特點比較：**

| 特性 | RISC-V PMP | ARM MPU | 建築學影響分析 |
| :--- | :--- | :--- | :--- |
| **最小區域大小** | 4 Bytes | 32 Bytes | 4位元組粒度允許對小型資料結構進行極細粒度控制，有效緩解緩衝區溢位。 |
| **最大區域數量** | 16 | 16 | 兩者在區域數量上相似，但 RISC-V 配置更靈活。 |
| **鎖定機制 (L bit)** | 設定後 M-mode 亦受限 | 獨立特權設定 | 當 L-bit 置位，即便 M-mode 也無法更改設定，**必須系統重置 (Reset)** 才能清除，提供強大的永續性安全保證。 |

### 5.2 標籤記憶體 (Tagged Memory) 與 Isolation 技術
以 **TIMBER-V** 為例，該架構為每個記憶體字（Word）增加標籤位元，實現細粒度的飛地（Enclave）隔離。
*   **N-tag：** 一般世界 (Normal World)。
*   **TU-tag/TS-tag：** 受信任世界 (Trusted World) 的使用者與監管者模式。
*   **TC-tag：** 標記受信任的進入點（Entry Point），防止非法跳轉攻擊。

### 5.3 資料隱身指令集擴充套件 (OISA)
為了徹底抵禦側向通道攻擊，研究提出了 **OISA (Data Oblivious ISA Extension)**。其核心為 **標籤站 (Label Station)** 機制，在執行單元中追蹤資料的 Public/Confidential 標籤，確保指令執行時間不隨運算元（Operands）而改變，從架構層面實現常數時間（Constant-time）執行。

## 6. 主要實驗設定與結果分析

### 6.1 處理器核心安全性實驗
研究人員在 Rocket Chip 生成器上針對不同核心進行了 Spectre 漏洞驗證：
*   **BOOM 核心：** 採用的「亂序執行 (Out-of-Order)」架構使其極易受到投機執行攻擊。實驗成功在 BOOM 上複製了 Spectre。
*   **緩解技術：** 實施 **L0 投機緩衝區 (Speculation Buffer)**，暫存投機載入的失敗資料，若確認為錯誤投機則立即清除，防止敏感資料影響快取狀態（Cache State），從而阻斷側向通道。
*   **Rocket 核心：** 作為「順序執行 (In-Order)」核心，其天然免疫大多數基於投機執行的攻擊。

### 6.2 效能基準比較
根據 Polarfire RISC-V FPGA (660MHz) 與 Xeon x86 伺服器處理器 (2.5GHz) 的密碼學運算效能對比：

| 運算型別 | Polarfire (KB/s) | Xeon x86 (KB/s) | 效能比值 |
| :--- | :--- | :--- | :--- |
| HMAC (MD5) | 32,421 | 572,497 | 5.66% |
| SHA256 | 7,392 | 386,888 | 1.91% |
| AES-256 CBC | 6,081 | 173,803 | 3.50% |

**分析觀點：** 儘管在絕對運算速度上 RISC-V 顯著低於 x86，但考慮到時脈頻率差異（0.66GHz vs 2.5GHz），嵌入式 RISC-V 在 **能效比 (Energy Efficiency)** 與 **單位面積效能** 上具有極大優勢。

## 7. 研究優點與限制

### 研究優點
*   **高效能基礎機制：** PMP 以極低的資源消耗提供強大的物理隔離能力。
*   **擴充套件靈活性：** 指令集擴充套件（ISE）能針對輕量化密碼學（如 Ascon）進行硬體加速，縮小與高效能處理器的差距。

### 研究限制與新興偵測技術
*   **物理攻擊威脅：** **電磁故障注入 (EMFI)** 實驗證明，攻擊者可透過 EMFI 導致 RISC-V 核心發生指令跳過或錯誤處理，這在低電壓環境下尤為危險。
*   **邏輯鎖定挑戰：** 為防範 IP 盜版，**SeqL (Logic-locking)** 技術被引入以隱藏正確金鑰，增加逆向工程難度。
*   **形式驗證缺口：** 雖然可審計性高，但偵測 RTL 臭蟲（Bugs）依然困難。研究提出了 **UPEC (Unique Program Execution Checking)** 形式化方法，系統性地偵測微架構層級的隱蔽通道。

## 8. 應用場景分析
根據研究資料（Table 1），RISC-V 的安全架構已實現在多項高效能領域：
*   **5G 與邊緣運算：** 如阿里巴巴 Xuantie-910。
*   **人工智慧 (AI)：** 結合運算加速與 PMP 許可權控制。
*   **智慧儲存控制器：** SiFive 核心已用於 SSD 控制器，管理金鑰加密。
*   **超級電腦：** 歐洲處理器倡議 (EuroHPC) 將 RISC-V 定位為其核心方案。

## 9. 與嵌入式系統安全之關聯
RISC-V 作為嵌入式安全的基石，透過 PMP 提供的強隔離與密碼學指令擴充套件（ISE）的效能提升，完美解決了資源受限環境中「高安全強度」與「低效能開銷」之間的權衡問題。透過減少 TCB 並建立硬體輔助的受信任執行環境（TEE），RISC-V 得以在物聯網節點上實現端到端的安全保障。

## 10. 結論與未來趨勢
RISC-V 安全生態系統正趨於成熟，其開放性促進了微架構安全研究的透明化。
未來發展趨勢包括：
1.  **密碼學指令集 (Crypto ISE) 標準化：** 統一純硬體密碼學擴充套件規範。
2.  **抗側向通道硬體化：** 將 OISA 等資料隱身技術引入高效能設計。
3.  **後量子密碼學 (PQC)：** 針對 SIKE 等演算法最佳化軟硬體協同設計。
4.  **全記憶體加密與驗證：** 從標籤機制邁向全透明記憶體加密，抵禦物理探針攻擊。
