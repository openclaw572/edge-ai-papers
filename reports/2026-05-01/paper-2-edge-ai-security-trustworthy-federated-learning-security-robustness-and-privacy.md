# 可信聯邦學習綜述：從安全性、魯棒性與隱私觀點出發

> 原始論文標題：A Survey of Trustworthy Federated Learning with Perspectives on Security, Robustness, and Privacy

## 執行摘要

值得信賴的聯合學習 (TFL) 代表了人工智慧的關鍵演變，旨在解決分散式環境中資料隔離和隱私的挑戰。隨著人工智慧技術擴充套件到金融、醫療保健和生命科學等安全關鍵領域，對本質上私密、可靠和對社會有益的系統的需求變得至關重要。

TFL 由三個基本支柱定義：
1. **隱私：** 確保個人資料的安全和私密處理，防止洩漏。
2. **穩健性：** 即使在極端條件或對抗性攻擊下也能保持穩定可靠的模型訓練。
3. **安全性：** 保證系統在部署過程中的機密性和正確性。

本檔案綜合分析了 TFL 在整個開發生命週期（資料處理、模型訓練和部署）中面臨的威脅，並評估了目前正在開發的用於緩解這些漏洞的技術防禦機制（SMPC、HE、TEE 和差異隱私）。

## 關鍵主題詳細分析

### 1. TFL 框架與開發生命週期
可信任聯邦學習由於具有資料隔離、非IID（獨立同分佈）資料分佈和通訊約束等獨特特徵，不同於傳統的集中式機器學習。該框架識別了每個開發階段的具體威脅：

|發展階段|主要威脅|描述 |
| :--- | :--- | :--- |
| **資料處理** |資料中毒，非獨立同分佈資料 |惡意行為者修改標籤或功能；客戶端之間的資料差異會減慢收斂速度。 |
| **模特兒訓練** |模型中毒、拜占庭錯誤 |參與者上傳設計的引數來降低準確度或植入隱藏的後門。 |
| **部署** |推理攻擊、查詢入侵 |攻擊者從已發布的模型或 API 推斷出敏感的訓練資料或模型屬性。 |

### 2. 安全與隱私的區別
該分析透過區分 TFL 環境中的安全性和隱私性，澄清了業界常見的困惑：
* **安全：** 關注系統和公共利益。它確保未經授權的使用者無法存取權重和梯度等內部資料。它保護計算過程的「機密性和正確性」。
* **隱私：** 關注個人。它保證即使攻擊者獲得了對內部 TFL 元件（權重、模型或梯度）的存取許可權，特定資料記錄或成員資格等私人資訊仍然受到保護。

### 3. 防禦的技術支柱

#### 安全聯合學習 (SFL)
SFL 利用安全計算來保護資料和模型的完整性。關鍵技術包括：
* **安全多方運算 (SMPC)：** 包含亂碼電路 (GC)、不經意傳輸 (OT) 和秘密共享 (SS)。雖然用途廣泛，但這些通常涉及高昂的溝通成本。
* **同態加密（HE）：** 支援對加密資料（密文）進行不解密的操作。它的範圍從部分 (PHE) 到完全同態加密 (FHE)。
* **可信任執行環境 (TEE)：** 使用硬體隔離（例如 Intel SGX）建立“飛地”，程式碼可以在其中私密且忠實地執行。

#### 穩健聯邦學習 (RFL)
RFL 專注於針對內部威脅的模型效能穩定性，特別是拜占庭問題（不可靠的客戶端）和有針對性的攻擊。
* **穩健聚合：** Krum、Bulyan 和 Trimmed Mean 等規則可減輕異常值的影響。
* **基於最佳化的方法：** FedProx 和 SCAFFOLD 等策略可解決由非 IID 資料引起的「客戶端漂移」。
* **叢集聯合學習 (CFL)：** 將具有相似資料分佈的使用者端進行分組，以提高專業模型的效能。

#### 隱私保護機制
* **差分隱私 (DP)：** 增加與單一個人資料的潛在影響成比例的隨機雜訊。這分為全域DP（伺服器新增的雜訊）和本地DP（客戶端新增的雜訊）。
* **擾動方法：** 將加性或乘性雜訊引入共享引數或本地資料集。
* **匿名化：** 使用方案（有時基於 GAN）來保護使用者級身份，而不會像 DP 那樣犧牲資料實用性。

## 帶有上下文的重要引言

> **「信任可以定義為『委託人和受託人之間的關係：委託人信任受託人。』在 TFL 的背景下，我們將受託人定義為 FL 模型或系統，將委託人定義為 FL 的參與者、使用者和監管者。」**

*背景：這句話奠定了 TFL 的哲學和營運基礎，將重點從純粹的技術指標轉移到多利益相關者關係模型。 *

> **「FL在Non-IID設定中的效能下降是不可避免的。區域性更新的發散不斷累積，減慢了模型收斂速度並削弱了模型效能。」**

*背景：在穩健性的討論中強調，這強調了在異質、分散的資料上訓練全域性模型的基本技術挑戰。 *

> **「隱私保證即使攻擊者可以存取內部TFL，如權重、模型、梯度等，隱私資訊仍可保留。」**

*上下文：這有助於闡明 TFL 的「縱深防禦」方法，其中隱私是系統安全受到損害時的最後一道防線。 *

## 可行的見解與未來方向

### 1. 效率與安全權衡的最佳化
目前的 SFL 演演演算法，特別是基於完全安全聯合訓練 (FSFT) 的演演演算法，面臨嚴重的延遲挑戰。未來的發展必須集中在**工程最佳化**，例如GPU加速運算（例如CryptGPU和Piranha）以及與主流機器學習框架更好的整合。

### 2.適應非獨立同分佈的現實
標準的拜占庭穩健方法在現實世界的 FL 中通常會失敗，因為它們無法區分惡意的「中毒」梯度和來自非 IID 客戶端的良性但「多樣化」的梯度。開發能夠識別對抗性攻擊的分佈變化的「自適應保護機制」是一個高度優先的研究領域。

### 3. 實作混合防禦策略
沒有任何一種技術解決方案可以提供絕對的可信度。最具彈性的 TFL 系統可能是**混合方法**，它結合了：
* **加密**以確保安全。
* **差異隱私**用於隱私。
* **強大的聚合**以確保穩定性。

### 4. 制定標準化基準
明顯缺乏統一的基準來驗證新的 FL 方法的穩健性和隱私權宣告。建立**標準化 TFL 基準**對於社群驗證針對複雜和不可預測威脅的防禦機制至關重要。

### 5. 邁向自適應保護
需要保護的資料和模型引數因應用場景而異。未來的SFL應該朝著**自適應保護**的方向發展，讓系統能夠在特定階段準確地只保護必要的元件，從而提高系統的整體效率。
