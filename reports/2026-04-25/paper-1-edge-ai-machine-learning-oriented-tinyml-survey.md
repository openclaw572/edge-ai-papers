# 機器學習導向的 TinyML 綜述

## 執行摘要

本檔案綜合了維羅納大學研究人員進行的一項全面調查，於 2023 年 9 月發表在 *IEEE Access* 上。該研究提供了基於 PRISMA 方法的系統性文獻回顧 (SLR)，涵蓋 2018 年 1 月至 2023 年 9 月的 TinyML 格局。

TinyML 被定義為一種正規化，使機器學習 (ML) 能夠在處理器和記憶體要求最低的邊緣裝置上執行，通常消耗的功率在幾毫瓦或更小範圍內。隨著現代神經網路擴充套件到數十億個引數，大規模的能源需求變得不可持續，這使得 TinyML 成為第四次和第五次工業革命的關鍵研究方向。該調查特別強調了機器學習的觀點——建議元學習、理性啟用函式 (RAF) 和多功能學習最佳化器方面的進步——同時詳細介紹了支援智慧邊緣應用程式的硬體和軟體生態系統。

## 關鍵主題詳細分析

### 1.TinyML 實作工作流程
該調查正式確定了開發 TinyML 解決方案的三個不同的工作流程，並按設計過程的起點對它們進行分類：

* **以機器學習為導向的工作流程：** 專注於固定或有限硬體平臺的模型設計、調整和訓練。包括模型設計、最佳化、上位評估、目標部署、最終評估等階段。
* **硬體導向的工作流程：** 專注於設計針對 ML 工作負載最佳化的增強型硬體平臺（例如 ASIC、FPGA）。這通常涉及建立硬體加速器以降低卷積等特定操作中的計算複雜性。
* **協同設計工作流程：** 機器學習專家 (G1) 和硬體專家 (G2) 從一開始就進行協作的整體方法。這種相互交織的過程允許透過硬體通知的訓練建立客製化架構（例如，模擬記憶體計算）。

### 2. 學習全景：最佳化與設計
TinyML 開發的核心在於平衡準確性與嚴格的資源限制（千位元組記憶體和毫瓦功耗）。

#### 模型最佳化技術
|技術|描述 |關鍵方法|
| :--- | :--- | :--- |
| **修剪** |消除冗餘權重或神經元以減少模型佔用空間。 |權重、神經元與結構化剪枝。 |
| **量化** |將高精度（32 位元浮點數）轉換為較低位寬度（例如 8 位元整數）。 |量化感知訓練（QAT）和訓練後量化（PTQ）。 |
| **知識蒸餾** |將知識從大型「教師」模型轉移到較小的「學生」模型。 |基於響應、基於特徵、基於關係；離線、線上和自蒸餾。 |
| **超引數最佳化 (HPO)** |自動搜尋最佳超引數（學習率、批次大小）。 |網格搜尋、隨機搜尋、貝葉斯最佳化。 |

#### 模型設計創新
* **神經架構搜尋 (NAS)：** 自動發現專門針對微控制器 (MCU) 的最佳架構，以滿足嚴格的記憶體和能源限制。
* **有理啟用函式 (RAF)：** 可訓練的啟用函式，可以透過使用更靈活的形狀對更廣泛的資料分佈進行建模來提高效能。
* **深度可分離卷積：** 透過將標準卷積分割為逐點和空間卷積（例如 MobileNet）來減少引數和操作。
* **注意力機制：** 使模型能夠專注於高維度資料的相關部分，從而改善資源受限環境中的上下文感知預測。

### 3.學習演演演算法與正規化
TinyML 研究正在從標準監督學習擴充套件到解決資料稀疏性和模型退化的正規化：

* **無監督學習：** 主要用於標記資料稀缺的工業、環境和醫療保健環境中的異常檢測。
* **自監督學習 (SSL)：** 一個新興領域，它使用代理任務來捕獲資料結構，無需手動標籤，對於物聯網中的異常檢測特別有用。
* **元學習：** 專注於“學會學習”，允許模型以最少的資料快速適應新任務。
* **持續學習：** 旨在透過允許部署的模型從新資料流中學習而不失去先驗知識來對抗「災難性遺忘」。技術包括基於正規化、基於重播和動態架構。

### 4. 硬體與軟體生態系統
該調查比較了 TinyML 中使用的主要硬體類別：

|硬體|優點 |缺點 |
| :--- | :--- | :--- |
| **CPU (Cortex-M)** |通用型、記憶體容量大、用途廣泛。 |並行度低，吞吐量低。 |
| **GPU (Nvidia Jetson)** |高吞吐量，非常適合併行執行/SOTA 架構。 |價格昂貴，耗能。 |
| **FPGA** |高度節能且靈活。 |使用困難，缺乏全面的函式庫。 |
| **TPU（珊瑚色）** |張量運算的推理表現大幅提升。 |價格昂貴，開發難度高。 |

#### 最先進的軟體工具
* **TensorFlow Lite Micro：** MCU 的開源框架；不需要作業系統或動態記憶體分配。
* **uTensor：** 針對 Arm 目標進行最佳化的極輕量級 C++ 函式庫。
* **Edge Impulse：** 用於訊號處理和訓練的基於雲端的 MLOps 平臺。
* **X-CUBE-AI：** STMicroElectronics 工具，用於將模型轉換為 STM32 最佳化程式碼。
* **uTVM：** 編譯器提供圖形級和操作員級最佳化以實現效能可移植性。

## 帶有上下文的重要引言

> **「TinyML 是一種範例，有助於在邊緣裝置上以最低的處理器和記憶體要求執行機器學習；因此，此類系統的功耗預計將在幾毫瓦或更低。」**

*背景：這提供了 2018 年建立的該領域的基本定義，強調了 TinyML 與標準 Edge AI 的區別的極端功率限制。 *

> **「執行這些網路所需的能量與其大小成正比，使得這種擴大神經網路的趨勢在大規模上不可持續。」**

*背景：這證明瞭 TinyML 研究的必要性。隨著標準機器學習模型達到數十億個引數，它們變得過於能源密集，無法廣泛部署，因此需要「更小型」的解決方案。 *

> **「TinyML 帶來了使它們可改革的工作，即允許 TinyML 在部署後從新資料點中學習的工作。這源於對抗模型漂移的需要。」**

*背景：這凸顯了該領域從僅靜態推理模型到能夠在裝置上學習以隨著時間的推移保持準確性的動態系統的轉變。 *

> **「TinyML 的 ImageNet 是什麼？」**

*背景：討論中提出的一個關鍵問題是，缺乏專門為 TinyML 感測器處理任務量身定製的大規模、普遍接受的公共資料集。 *

## 未來方向的可行見解

根據調查對未探索領域和當前侷限性的分析，為從業者和研究人員確定了以下方向：

* **優先考慮協同設計：** 擺脫孤立的機器學習或硬體工作流程。協同設計方法透過使“形式和內容具有可塑性”，為效能突破提供了最大的希望。
* **探索自監督和弱監督學習：** 這些領域目前在 TinyML 中代表性不足。開發這些可以減少對昂貴的手動標籤的依賴，這是邊緣應用程式的主要瓶頸。
* **解決基準測試差距：** 迫切需要公認的基準測試，同時考慮硬體異質性、軟體堆疊和功耗，以便對解決方案進行科學比較。
* **投資資料驅動工程：** 除了模型準確性之外，未來的開發必須優先考慮「資料卓越」。需要工具來評估資料品質並瞭解現實場景中的特定模型故障點。
* **解決記憶體碎片：** 由於 TinyML 在 SRAM 和快閃記憶體極小的裝置上執行，研究人員應專注於頻寬分配的啟發式技術和更有效的數位表示（例如，使用少於 8 位元），而不影響準確性。
